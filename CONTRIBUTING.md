# Contribuir

RAVENLOCK debe mantenerse local, defensivo y auditable.

Reglas:

- No agregar funciones ofensivas.
- No agregar telemetría.
- No borrar archivos automáticamente.
- No ocultar procesos.
- No modificar el sistema sin confirmación.
- Mantener reportes claros.
- Mantener el código simple de auditar.
